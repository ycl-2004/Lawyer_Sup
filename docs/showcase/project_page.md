# LawDesk Junior — 给法律 AI 装上「不许猜」的工程约束

**一句话**：面向初级律师的劳动争议仲裁工作台——把散乱的合同/通知/聊天记录，整理成
**带来源、可复核、缺失不补全**的仲裁文书草稿；核心不是「AI 多聪明」，而是「**AI 不许乱猜**」是如何被做成可验证的工程约束的。

> ⚠️ 内部辅助工具 · 所有输出均为草稿 · 不构成法律意见 · 不评估胜诉 · 须承办律师逐项确认。

## 为什么是它（差异点）

市面上的「法律 AI」大多是把材料喂给大模型让它自由生成——**好看但不可信**：会编金额、会幻觉法条、会把没有的事实写得言之凿凿。
本项目反过来，把**风控做成第一性约束**：

| 主张 | 代码 | 测试 | UI 行为 |
|---|---|---|---|
| 金额不靠模型猜 | `compensation_service.py` / `compensation.ts` | 30+ golden/边界用例 | 缺输入→卡片「无法计算」 |
| 事实必须有来源 | `fact_extractor.py` | 评估集 引用有效率 100% | 每个字段挂原文引用 |
| 引用不许幻觉 | `citation_check.py` | golden 抓幻觉法名/条号 | 复核页标红高风险 |
| 律师说了才算 | `reviewGate.ts` | 7 项门槛用例 | 未确认→导出锁死 |

**四层一一对应**——这条「主张→代码→测试→UI」的证据链，是抵御深度追问的底气。

## 技术架构（30 秒）

- **前端** React + Vite + TS + Tailwind；**后端** FastAPI（stdlib 优先，mock 默认离线）。
- **真实链路**：上传 PDF/docx/txt → 解析 → 规则抽取（+ 可选 LLM 增强，引用逐字校验）→
  确定性计算 → 法条 RAG（BM25+TF-IDF+关键词）→ 模板草稿 → 复核门槛 → 带水印导出。
- **持久化** 本地 SQLite（14 天 TTL）；**单地址运行**（后端托管前端，双击启动）。
- **双实现一致性**：金额计算 TS/Python 两份，共享同一套用例，CI 双侧守护防口径漂移。
- 详见 [架构图](../architecture.md)。

## 可验证的成果（本机 2026-06-21 实测）

- 前端 **41** vitest + typecheck + build；后端 **50** pytest，全绿（[存档](../acceptance/test_output_archive.md)）。
- 抽取评估（规则层，离线，诚实记录）：**召回 84% / 精确 100% / 编造 0 / 来源引用有效率 100%**。
- 端到端：全新「陈某」案 → 真实 PDF 解析 → 抽取 7 字段 → 律师确认 → **重启后端数据仍在**。
- 评估集还**当场抓出 2 个真实抽取 bug**（解除条款误判入职日期、引用非原文子串），已修复。

## 诚实的局限（主动声明 = 加分）

- LLM 真实抽取**已接线但质量未评估**；离线评估只覆盖确定性规则层。
- 法条为**人工整理演示样例**，正式引用须核对官方文本（[追踪表](../legal/corpus_verification.md)已就绪）。
- RAG 是 BM25+TF-IDF（**非稠密向量**），接口已为 BGE-M3 预留；语料 20 条，小而精。
- 无登录/加密/鉴权；接真实客户材料前需补齐 PIPL 合规。
