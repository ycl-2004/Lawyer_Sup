# 演示脚本（3–5 分钟口径稿）

> 目标：用最短路径展示本项目区别于「又一个法律聊天机器人」的核心——
> **确定性计算 + 来源锚定 + 反幻觉核对 + 律师确认门槛**这套「不许猜」工程体系。
> 先读最后一页「诚实声明」，全程口径与之一致。

## 开场（30 秒）

> 「LawDesk Junior 是给**初级律师/律师助理**用的**内部辅助工作台**，做劳动争议仲裁材料整理。
> 它的定位不是替律师下判断，而是把散乱材料整理成**可复核的草稿**——所有输出都标注草稿、附来源、可被律师逐项否决。
> 我要演示的重点不是‘AI 很聪明’，而是‘**AI 不许乱猜**’是怎么用工程约束实现的。」

## 主线一：实时案件全链路（90 秒）

1. Dashboard →「新建案件」→ 填当事人代号与对方单位。
2. 上传一份劳动合同 + 解除通知（PDF/txt）。→ 「这是**真实解析**，不是预置数据。」
3. 「运行 Agent 工作流」→ 指着弹窗：「逐节点真实执行，绿色徽章说明走的是**后端 FastAPI 管线**。」
4. 工作区出现抽取事实 → 指着来源引用：「**每个事实都挂着原文逐字引用**；引用不在原文里会被强制标红复核。」
5. **刷新页面** →「数据还在——本地 SQLite 持久化，不是演示态内存。」

## 主线二：「不许猜」红线（90 秒，最重要）

6. 打开**刘某案（加班费）**：加班小时为空 → 计算卡片显示**「无法计算 — 缺少输入」**。
   > 「换成聊天机器人，这里会编一个数字。我们的红线是：**缺输入就拒绝计算**，绝不从打卡记录推测工时。」
7. 手动填入核定小时数 → 金额立即算出，并展开**推导过程 + 法条依据**。
   > 「公式写死在代码里，LLM 不参与计算；每一步可见、可复核。」
8. 复核页「核对全部草稿引用」→ 抓出《不存在法》/第九百条等**幻觉引用**标红。
   > 「草稿里只要引了知识库里不存在的法条，**反幻觉核对会当场抓出来**。」
9. 指着导出门槛：「高风险发现没处理完、确认清单没勾、没填律师名——**确认与导出按钮是锁死的**。」

## 收尾（30 秒）

10. 导出 docx → 首页红色「草稿」水印；导出材料包 → 含免责声明 + 确认审计 + 脱敏。
    > 「从主张到代码到测试到 UI 行为，**四层一一对应**，这是这个项目最想展示的东西。」
11. 命令行一闪：`pytest`（50）、`npm test`（41）、`run_eval`（编造 0 / 引用有效 100%）。

## 诚实声明页（务必先读，回答追问的底气）

**可以放心说（有代码+测试支撑）：**
- 确定性金额计算（TS/Python 双实现，golden + 边界用例，CI 守护）。
- 反幻觉引用核对（能抓幻觉法律名与幻觉条号，有 golden）。
- 混合检索 BM25+TF-IDF+关键词（golden 10/10）。
- 真实上传解析 + SQLite 持久化（本机端到端实测：全新案件抽取 7 字段、重启不丢）。
- 规则抽取评估：召回 84% / 精确 100% / 编造 0 / 引用有效 100%（诚实记录，仅规则层）。

**必须保守表达（接线就绪 ≠ 已验证）：**
- 「DeepSeek 真实抽取**已接线**，但其抽取**质量未经评估**」——别说「已接入并验证」。
- 「法条为**人工整理演示样例**，正式引用**须核对官方文本**」（追踪表已就绪）。
- 「RAG 为 BM25+TF-IDF（**非稠密向量**），接口已为 BGE-M3 预留」——主动说明反而加分。
- 「20 条语料**体量小**」——强调「小而精 + 元数据完整」的取舍。

**绝对不能说：**
- ❌「AI 自动完成案件分析」「可投入律所使用」「已接入 DeepSeek 实际运行并验证」
- ❌ 任何暗示**替代律师**、**预测胜诉**、**法条库权威完整**的表述。

**一句话定位：** 「这是一套面向律师内部的**辅助草稿工具**，核心价值是**把‘不许猜’做成可验证的工程约束**；
它不出法律意见、不评估胜诉、所有成果须承办律师确认。」
